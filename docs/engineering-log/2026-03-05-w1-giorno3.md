# W1 - Giorno 3
Data: 2026-03-05

## Focus

Containerization, frontend integration and deploy readiness.

L'obiettivo della giornata è stato portare il progetto da una configurazione di sviluppo locale a un'architettura containerizzata e pronta al deploy.

## Activities

### Dockerization

Sono stati creati Dockerfile dedicati per:

- backend FastAPI
- frontend Next.js

Il frontend utilizza un build multi-stage per generare una build standalone.

### Docker Compose

È stato introdotto un file `docker-compose.yml` nella root del progetto per orchestrare i servizi.

Funzionalità introdotte:

- networking automatico tra container
- esposizione porte
- caricamento environment variables
- gestione restart policy

### Healthchecks

Sono stati implementati healthcheck per entrambi i servizi.

Backend:

endpoint `/health`

Frontend:

HTTP request alla root application.

Questo permette a Docker di monitorare lo stato dei container.

### Debug e stabilizzazione

Durante l'integrazione sono stati affrontati diversi problemi:

- path del server standalone di Next.js
- gestione IPv4 vs IPv6 nei healthcheck
- configurazione corretta del runtime server
- gestione delle dipendenze nei build multi-stage

Il sistema è stato stabilizzato fino a ottenere stato `healthy` per entrambi i container.

### Hardening

Sono state applicate alcune misure di sicurezza e stabilità:

- introduzione `.dockerignore`
- esecuzione container come utente non-root
- limiti CPU e memoria nei container
- immagini base minimali

## Result

Il progetto è ora completamente containerizzato e orchestrato tramite Docker Compose.

L'architettura è pronta per il deploy su piattaforme container-based.

Servizi attivi:

Frontend → http://localhost:3000

Backend → http://localhost:8000
